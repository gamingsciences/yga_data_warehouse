dbo.CP_PrintTable 
Logical Entity Name  CP_PrintTable 
Default Table Name  CP_PrintTable 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  This is table is used to PRINT COUPONS It is intended that this table be used by a crystal report template. Therefore the crystal will control what is to be printed. For example there is information about the player and his address. The crystal can print the information in this field or simply chose to ignore it. ie Most of the time they may NOT wish to print player information, but it is here for maximum flexibility.

dbo.CP_PrintTable Columns 
ColumnName  Domain  Datatype  NULL  Definition 
Offer_ID    int  YES  Linked to the CP_Offer Table. 
Number    int  NO  Just a humble record counter. If there are 820 records in this table. This runs from 1 to 820. 
PR    int  YES  PR and CP are counters that work together. If there are to be 4 offers and 3 coupons in each one PR and CP would run like this:
11 12 13 21 22 23 31 32 33 41 42 43 
CP    int  YES  See PR. 
Player_ID    int  YES  Unique ID number given to each player added via the Player Notebook in Super-PlayMate or PitBOSS.

LastName    varchar(255)  YES  Last Name of the Player. 
AddressLine1    varchar(255)  YES  Address Line 1 of the Player. 
AddressLine2    varchar(255)  YES  Address Line 2 of the Player. 
City    varchar(100)  YES  City of the Player. 
State    varchar(100)  YES  State of the Player. 
Zip    varchar(80)  YES  Zip code of the Player. 
Country    varchar(200)  YES  Country of the Player. 
CouponName    varchar(100)  YES  Name of the Coupon. This comes from the table CP_Coupon. 
CouponDesc    varchar(255)  YES  Coupon Description from the table: CP_Coupon. 
CouponType    varchar(100)  YES  Coupon Type. This comes from the table CP_Coupon. 
CouponSubType    varchar(100)  YES  Coupon SubType. This comes from table: CP_Coupon. 
ValueM    money  YES  The value of the Coupon. Note: the reason we use ValueM is becasue Value is a "keyword." 
SpelledOut    varchar(200)  YES  The ValueM spelled out like on a bank check. For example, 103.46 is "one hundred three dollars and forty six cents." 
StartDate    varchar(120)  YES  Coupon Start Date. This comes from the table CP_Coupon. 
EndDate    varchar(120)  YES  Coupon End Date. This comes from the table CP_Coupon. 
GLDebitCode    varchar(100)  YES  Coupon GL Code, originating from the CP_Coupon table. 
TicketNumber    varchar(100)  YES  Coupon TicketNumber. This comes from the table: CP_Coupon_Players_Validation. 
Prize    varchar(200)  YES  This is from the table: Cp_Coupon or for Imported Coupons from CP_Coupon_Players_Validation. 
Validation    varchar(36)  YES  Coupon Validation Number this comes from the table: CP_Coupon_Players_Validation. 
BarCode    varchar(100)  YES  This is a STRING used to print the barcode. It assumes that the Cyrstal Report using this has a special font from ELFRING software. The report can use this field OR it can print the bar code from the Validation Number using its own method. 
IssuedDate    varchar(120)  YES  Date Coupon was issued
