i_DB_OfferStatus 
Base Table  CP_Offers 
Attribute Name  i_DB_OfferStatus 
ColumnName  i_DB_OfferStatus 
Logical Rolename   
RoleName   
Domain   
Physical Only  NO 
Definition  A Very IMPORTANT FIELD. This field controls the STATUS of the Offer as it goes through the IMPLEMENT process. If the user starts the Implementation Process, it prevents the user from Editing.
0 = User can edit the Offer, and its Coupons (Exception Imported Coupons can Never be Edited).
100 = The User has chosen and reserved the Players who will receive the Offer but the Offer has yet to be finalized.
200 = Validation Numbers have been chosen for the Offer.
300 = Offer is PRINTED.
400 = Offer is Finalized, and is ready for the coupons to be cashed.
-999 = Emergency Void; All coupons in the offer are voided.

Note: Until the Offer reaches stage 400, the user can reverse the process. For example, you can go from Step 200 to 100. Once you reach stage 400, you can not go backwards. The reason for this is once records are written to the tables (e.g.CDS_Ticket), we do not want to ERASE them. They may be VOIDED, but not ERASED.
