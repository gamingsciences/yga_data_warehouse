AllowPitPaymentsAtOtherTable    char(1)  NO  Indicate if a Payment on a marker can be accepted on a table other than where the marker was issued from. 'Y' = Yes, 'N' = No. 
MarkerBalanceOnWithdrawal    smallint  NO   
CloserShift_ID    char(1)  YES  The last shift for a day which when crossed goes in to the next day.

LastSoftCountGamingDate    smalldatetime  YES  Gaming Date when last Soft Count was done. 
LastSoftCountShift_ID    char(1)  YES  Shift when last soft count was done. 
LastSoftCountPosted    char(1)  YES  Indicating if Soft count was posted to all the revenue tables. 'Y' = Yes, 'N' = No. 
LastGamingDateClosed    smalldatetime  YES  Indicates the last date when Gaming was closed. No transaction is allowed once gaming date is closed. 
LastDropClosedDate    smalldatetime  YES  Date when all revenue and counts were posted to the revenue tables.

DefaultMarkerDepositDays    int  NO  Number of days allowed for a marker to be deposited. 
ClearDays_Local    int  NO  Number of days given for a local check to clear. 
ClearDays_NonLocal    int  NO  Number of days given for an out-of-town check to clear. 
TTOSupercedes    char(1)  NO  Indicates if a credit limit is allowed to be increased or overridden. 'Y' = Yes, 'N' = No. 
ComboCredit    char(1)  NO  For an account having a primary and secondary player, this flag allows sharing of the credit. 
FIFOPayments    char(1)  NO  Indicates if a payment is allowed to go against the first marker issued. 
DefaultBankLocation_ID    int  YES  Indicates if Fills in PitBOSS are allowed. 
RollOverTime    smalldatetime  NO  Contains the same value as Close_shift_date.

Reportable    int  NO  The limit of what's reportable to the IRS.

ValidateIDQckChk    char(1)  YES  Allows a player's check to be accepted with only an ID check. 
Collections_Enabled    char(1)  NO  Is not currently used. Allows for Collections to be a part of the application. 
Muti_CageWindow_Roll    char(1)  NO  Flag allowing all cage windows to be rolled over at the same time. 
Print_Payment_Receipts    char(1)  NO  Flag allowing Payment Receipts to be printed from the Pit, Cage, or both. 
VoidCageDocs_ClosedGameDate    char(1)  NO  Indicates if the user is allowed to void cage documents generated on gaming days where the audit has closed the gaming day.
