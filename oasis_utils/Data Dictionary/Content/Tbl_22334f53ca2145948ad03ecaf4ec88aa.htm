dbo.IndexingHistory 
Logical Entity Name  IndexingHistory 
Default Table Name  IndexingHistory 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  Contains historical information from reindexing using the dbo.usp_Reindex procedure. This historical information is used for the reindexing logic to adjust the fill factors for indexes.

IndexingHistoryID    int  NO  Uniquely identifies a record in the IndexingHistory table. 
Object_ID    int  NO  Uniquely identifies the table of an index that was rebuilt and/or reorganized.

Rebuild    bit  NO  Flag indicating if the index was rebuild. If it wasn't rebuilt then it was reorganized. 
LeafPageCount    int  NO  Number of pages in the leaf level of the index. 
LeafFragmentCount    int  NO  Number of fragmented pages in the leaf level of the index before it was rebuild/reorganized. 
LeafLogicalFragmentation    float  NO  Logical fragmentation of the leaf level of an index before it was rebuild/reorganized. 
ScanDensity    float  NO  The percent of allocated space, that actually contains data for the index before it was rebuilt or reorganized. 
IndexFillFactor    tinyint  NO  The fill factor for the index before it was rebuilt or reorganized. 
NewFill    tinyint  NO  The new fill factor used during the rebuild of the index. 
InsertedDatetime    datetime2(7)  NO  Datetime the index started to be rebuilt or reorganized.
 
FinishedDatetime    datetime2(7)  YES  Datetime the index rebuild or reorganized finished.
 
Statement    nvarchar(max)  NO  The statement used to rebuild or reorganize the index.
