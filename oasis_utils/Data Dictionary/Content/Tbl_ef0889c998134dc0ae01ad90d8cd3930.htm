ColumnName  Domain  Datatype  NULL  Definition 
BatchType    char(1)  YES  Timestamp representing the time sends the keypad signal for the slot machine keypad. 
AddedByTracker    tinyint  YES  Denotes the source of the rating. Ususally contains the value of '1'. However, it could also be used to show ratings entered manually or via Gateway. 
SlotBatch_ID    int  NO  Unique Identifier ID for each record in the table. 
Player_ID    int  YES  Unique ID number given to each player added via the Player Notebook in Super-PlayMate or PitBOSS. 
OCR    char(10)  YES  10 digit number that uniquely identifies a player or employee card in the CDS_Card table.  
StatType    char(5)  YES  This indentifies the type of rating. 'Pit' and 'Slot' are the standard defualts, however additional types such as "Bingo", "Keno" or "SportsBook" can be added. 
Location    char(2)  YES  Casinos divide the gaming floor in to different areas. Each area is then divided into small unites called sections. Each section is further divided into small units called locations. Each location is then given a unique identifier. 
Pit    int  YES  The Pit location where the rating was performed. Used for pit ratings. 
TableNumber    int  YES  Unique number representing a specific game table. 
SlotNumber    int  YES  Slot Number refers to the number of the gaming machine. This number corresponds to the number on the machine's cabinet. 
StartTime    datetime  YES  The date and time when the player began playing the game. 
EndTime    datetime  YES  The date when the player finished playing the game. 
TimePlayed    float  YES  The total amount of time in seconds that the player played the game. 
PlayerSkill    tinyint  YES  The rated skill of the player: Soft, Medium, and Hard. These values are setup in OASIS Administrator. 
PlayerSpeed    tinyint  YES  The rated speed the player was playing: Slow, Medium, or Fast.These values are setup in OASIS Administrator. 
Denomination    numeric(12, 2)  YES  The monetary value a slot machine is setup to use. 
MoneyPlay    numeric(12, 2)  YES  Used by PitBOSS mainly to identifiy the amount of money for game play brought by the player to the table game. 
RawAverageBetTimesTime1    money  YES  The manually-entered betting average of the player that was made on one hand. 
RawAverageBetTimesTime2    money  YES  The manually-entered betting average of the player that was made on a second hand. 
RawAverageBetTimesTime3    money  YES  The manually-entered betting average of the player that was made on a third hand. 
CashIn    numeric(12, 2)  YES  The total amount of cash that the player used to play the game. 
FrontIn    numeric(12, 2)  YES  Used by PitBOSS mainly to identifiy the amount of front money the player used at the table game. 
ChipsIn    numeric(12, 2)  YES  Use by PitBOSS mainly to identifiy the dollar amount of chips brought by the player to the table game. 
CreditIn    numeric(12, 2)  YES  Used by PitBOSS mainly to identifiy the dollar amount of credit used by the player at the table game. 
CashOut    numeric(12, 2)  YES  Used to identifiy the total amount of money in hand with which the player walked away from the game. 
Jackpot    numeric(12, 2)  YES  Identifies the amount of any jackpots hit during the players session (card in to card out). 
Twin    money  YES  Theorical Win. This calculation is based on a number of vairables:
Pit TWin = TmePlayed * Speed * AvgBet * Skill
Speed: Fast, Medium, or Slow.
Slot TWin = Coinin * Par [Hold Percentage] 
PtsEarned    int  YES  The number of Points Earned based on the player's activity. 
PromoPtsEarned    int  YES  Points earned during promotional events. 
DollarsEarned    money  YES  Dollars Earned is the dollar amount of the Pts Earned, based on the Points per Dollar setting in OASIS Administrator. 
PromoDollarsEarned    money  YES  Promo Dollars Earned is based on a promo points-to-promo dollars calculation in OASIS Administrator. 
CompEarned    money  YES  Comp Earned is a calculated based on the comp calculation in OASIS Administrator. 
GamingDate    date  YES  The gaming date when the transaction was done. 
Shift_ID    char(1)  YES  Current cage shift. 'D' = Day, 'G' = Grave, 'S' = Swing. 
Local    char(1)  YES  Identifier that shows whether or not the player is a local (from the same town where the casino is located). 
PromoPlay    money  YES   
User_ID    int  YES  Unique system-generated ID number given to each casino user (employee). 
RatedByUser_ID    int  YES  The User ID who issued the rating. 
DeltaSessions    int  YES  Extra columns to be used later. Not used currently.

DeltaDenomSessions    int  YES  Extra column to be used later. Not used currently. 
DeltaPointCalcSessions    int  YES  Extra column to be used later. Not used currently. 
PointCalc_ID    int  YES  Unique ID number used to identify the Points/Comp calculation. 
Account_ID    int  YES  Player Account Number. 
IsSuspendedRating    bit  NO  Flag indicating if the record is a suspeneded rating. An example of a suspended rating is one that was in violation of the Max Card Ins limit. 
IDType    AS ('P')    Previously indentified the account type. 'A' = Account, 'P' = Player, 'G' = Group. in version 12.4 and forward, only type P will be used. This column was maintained for backward compatibility and changed to a computed column where P will always be inserted as the value. The table scripts will only keep type P records on during an install/upgrade.
