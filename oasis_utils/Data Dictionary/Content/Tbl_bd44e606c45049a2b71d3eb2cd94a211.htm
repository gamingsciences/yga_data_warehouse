dbo.CP_Coupons Columns 
ColumnName  Domain  Datatype  NULL  Definition 
Offer_IX    int  NO  This is a reference to the go with Offer_ID. This indicates with what Offer this coupon is associated. There may be many offers for one campaign. 
Coupon_ID    int  NO  This is the identity for this table. Referenced by several other tables. 
sCouponName    varchar(50)  NO  The Name of Coupon. Campaigns are created to hold Offers which in turn hold individual Coupons. 
sCouponDescription    varchar(500)  YES  A brief description of the Coupon. This is entered from Appliction Coupons in from frmCampaigns. 
dateCouponStart    datetime  YES  This is the date that the Coupons Starts. After the Start Date, the casino will accept the Coupon until the dateCouponEnd. 
dateCouponEnd    datetime  YES  This is the Date that the Coupons End. After the Start Date, the Casino will accept the Coupon until the dateCouponEnd. 
PrizeOrComp_ID    int  YES  This is an ID into the TABLE CDS_PRIZE. It states what Comp or Prize the user is given by this coupon. 'NA' if it is a cash coupon. 
sPrizeOrCompDescription    varchar(50)  YES  States what Comp or Prize the User is given by this coupon. 'NA' if it is a cash coupon. For the most part the other variable Prize_Or_Comp_ID is used. 
sngCost    money  YES  The COST of a Prize or Comp. In the case of a Cashable Coupon, sngCost and MaxValue are equal. 
sngMaxValue    money  YES  The COST of a Prize or Comp, in the case of CASH simple the Cash Value. In the case of a Cashable Coupon sngCost and MaxValue are equal. 
iActiveCoupon    smallint  YES  Unused field. 
iCashableCoupon    smallint  YES  Unused field. 
iUniqueNumbers    smallint  YES  VERY IMPORTANT FIELD. Indicates if a series of Coupons in an Offer have different valication numbers. If True, then all the Coupons will have different validation numbers. If False, then all the Coupon validation numbers are the same.This affects A LOT in the COUPONS program: in which table the Coupons are stored, how they are REDEEMED and VOIDED, and so on. 
dateCouponGenerated    datetime  YES  NOT USED. 
sCouponType    char(1)  YES  Indicates the type of the Coupon. 'S' = Slot, 'M' Marketing. See sCouponSubType. 
sGL_Debit_Code    char(18)  YES  A letter code printed on the front of every coupon in the series. 
sCouponSubType    char(5)  YES  Works with sCouponType to Describe the Coupon. If sCouponType = M then sCouponSubType can be Prize, Comp, or Cash. If sCouponType = M then sCouponSubType can be Prom, Cash, or Promotional. See sCouponType. 
IssuedID    int  YES  The ID of the user who created the coupon. 
IssuedAppID    int  YES  This is the application ID. 
IssuedDateTime    datetime  YES  ISSUED is no longer recored when anything is writyen to the Coupon Tables. 
IssuedLocation_ID    int  YES  Terminal location from where the Coupon was issused. 
PrintedID    int  YES  Holds the OASIS User ID of the person who printed this Coupon (series). 
PrintedAppID    int  YES  The ID of the Application that printed the Coupon, most likely the Coupon Application. 
PrintedDateTime    datetime  YES  Timestamp of exactly when a transaction was printed. 
PrintedLocation_ID    int  YES  Location of the PC that perfomed the PRINT operation. 
VoidedID    int  YES  The OASIS User ID of the person who voided the Coupon. 
VoidedAppID    int  YES  The ID of the Application that voided the Coupon, most likely the Coupon application.. 
VoidedDateTime    datetime  YES  Timestamp of exactly when a transaction was voided. 
VoidedLocation_ID    int  YES  Location of the PC that perfomed the Void Operation. 
iImported    int  YES  Indicates if the Coupon was imported. True or False. 
iSequence    bigint  YES   
iIndividualCouponStatus    int  NO  It was decided that Individual Coupon series within an Offer could be voided. (This is different than voiding an individual Coupon.) For example, say they print out an offer to 162 players, with 3 coupons. One of the coupons was supposed to be for 10 dollars, but a misplaced decimal made it for 1000 dollars, then you can void out all these coupons. Note: 2 = PRINTED, 6 = VOIDED. 
RedeemTimeFrame    tinyint  YES
