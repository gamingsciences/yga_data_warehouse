dbo.RevenueItem 
Logical Entity Name  RevenueItem 
Default Table Name  RevenueItem 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  Lookup table for revenue items. A Revenue item such as VoucherIn can have one or more Revenue Values such as VoucherInSystem, VoucherInActual, or VoucherInMetered. Some are actually virtual or calculations of other Revenue Items.

dbo.RevenueItem Columns 
ColumnName  Domain  Datatype  NULL  Definition 
RevItemID  RevItemID  tinyint  NO  Uniquely identifies a Revenue Item from the RevenueItem table. A revenue item is a unqualified revnue value such as Bill In, which has 3 more specific Revenue Values of Actual Bill In, Metered Bill In, and Sytsem Bill In.  
Name    varchar(255)  NO  Name of the Revenue Item. 
DetailOfID    tinyint  YES  DetailOfID is a RevItemID of the parent if this revenue item is related to another. For example the Revenue Item Ones, Twos, Fives, .... are the childern of Bill In. 
MeterID    tinyint  YES  Uniquely identifies a meter. This is not necessarily the same identifier that the sentinel uses which is SenMeterID.  
MtrRevValueID    tinyint  YES  MtrRevValueID is a RevValueID of the Metered Revenue Value for this Revenue Item. 
ActRevValueID    tinyint  YES  ActRevValueID is a RevValueID of the Actual Revenue Value for this Revenue Item. 
SysRevValueID    tinyint  YES  SysRevValueID is a RevValueID of the System Revenue Value for this Revenue Item. 
IsMultiGame    bit  NO  Flag to indicate if this revenue item is for a multi game sub game. 
TypeFlags    tinyint  NO  TypeFlags is a bitmask to indicate which logical groups this revenue item belong too. The following are the groups and bit value: Bill = 1, AttendantPaidPayout = 2, Voucher = 4, Coupon = 8, PBT = 16. 
SortOrder    int  NO  Default order in which revenue items should appear in. 
CalculationOrder    tinyint  YES  The order in which all calculated RevenueItems should be calculated since some calculated values are dependant on other calculated values. 
Formula    varchar(4000)  YES  If this revenue item is calculated not a base value this is the formula used to calculate it.
