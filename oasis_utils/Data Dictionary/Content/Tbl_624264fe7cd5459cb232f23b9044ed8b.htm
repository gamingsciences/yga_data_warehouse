Default Table Name  CP_Campaigns 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  This table holds information for the Campaigns. Campaigns are created to hold Offers, which in turn hold individual Coupon Specifications. Campaign_ID is the Primary Field for this table; entries are referenced by this key. This data is entered on the Form frmCampaigns in the Coupons application.

dbo.CP_Campaigns Columns 
ColumnName  Domain  Datatype  NULL  Definition 
sCampaignName    varchar(50)  YES  The Name of the Campaign. Campaigns are created to hold Offers which in turn hold individual Coupon Specification. This is the NAME of the Campaign. This data is entered on the Form frmCampaigns in Application Coupons. 
sCampaignDescription    varchar(1000)  YES  The Description of the Campaign. Campaigns are created to hold Offers which in turn hold individual Coupon Specification. This is a brief DESCRIPTION of the Campaign data is entered on the Form frmCampaigns in Application Coupons. 
Campaign_ID    int  NO  This is the identity for this table. 
iImported    int  NO  Indicates if the Campaign was imported. True or False.  
DateCreated    datetime  NO  The date that the Campaign was first created. 
iUserWhoCreated    int  YES  The ID of the OASIS User who created this Campaign. 
sUserWhoCreated    varchar(50)  YES  This is the STRING of the User who Created this Campaign. This comes from iUserWhoCreated 
DateUpdated    datetime  YES  This is the date that this Campaign was Updated. Note: If any Offers or Coupons UNDER THIS CAMPAIGN are added or Changed, then the CAMPAIGN is considered to be updated. The Program uses DateUpdated and iUserWhoUpdated to avoid conflicts. Before a change is made to a Campaign (or underlying Offer or Campaign) the program makes sure that a no one else has made a change to the campaign since the last time data was read. If there was a change, it stops. If there wasn't a change, it makes the change, then sets these 2 values. 
iUserWhoUpdated    int  YES  The OASIS user who last updated and item per the comment in field DateUpdated. 
sUserWhoUpdated    varchar(50)  YES  The OASIS User (in string form) who last updated. See comment under field DateUpdated
