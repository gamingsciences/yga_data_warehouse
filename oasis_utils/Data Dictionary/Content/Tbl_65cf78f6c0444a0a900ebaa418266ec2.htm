Unmerge.StatDenom 
Logical Entity Name  STATDENOM 
Default Table Name  StatDenom 
Physical Only  NO 
Do Not Generate  False 
Owner  Unmerge 
Definition  This is a 3-part operation to preserve the records of a player that was merged with another player. The first part is for Slot ratings, the second part is for Pit ratings, and the thrid part is for all other ratings. For Slot ratings, the table is updated based on PlayerID, StatType (where the value is 'Slot'), and Denomination. If any of these conditions do not exist, then a row is inserted in the table. For Pit ratings, the table is updated based on PlayerID, StatType (where the value is 'Pit'), and TableType. Again, if any of these conditions do not exist, then a row is inserted in the table. For other ratings, the table is updated based on PlayerID, StatType (where the value is neither 'Slot' nor 'Pit'), and Outlet. If any of these conditions do not exist, then a row is inserted in the table. These records were recorded prior to the merge to facilitate an unmerge.

The table scripts will only keep type P records during an install/upgrade. The column may be removed in a future release.

The column was kept for backwards compatibility and may be removed in a future release.
