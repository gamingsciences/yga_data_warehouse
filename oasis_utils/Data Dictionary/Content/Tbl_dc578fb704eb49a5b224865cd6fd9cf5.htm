dbo.PB_Location 
Logical Entity Name  PB_LOCATION 
Default Table Name  PB_Location 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  Stores all information about a location and places where player transactions occur (i.e. pits, cage windows, and fill banks).

TimeStamp    datetime  YES  The date and time when the record was inserted or last updated. 
Casino_ID    tinyint  NO  Unique casino ID used in conjunction with mulit-casino or multi-site casinos. Default: dbo.DEF_CASINOID. 
Name    char(30)  NO  Name of the location. 
Desc_ID    int  NO  Descriptive IDs from PB_LOCATIONTYPES. 
Location_ID    int  NO  The location ID where the transaction was performed. 
GridLevel    int  NO  Level of the location in the hierarchy. 
ParentLocation_ID    int  NO  The parent's ID in the hierarchy. ID number of the location one grid level above the currently selected location. 
ChipLedger    char(1)  NO  Indicates if the location is avalilable to the chip ledger. 'Y' = Yes, 'N' = No. 
CashLedger    char(1)  NO  Indicates if the location is available to the cash ledger. 'Y' = Yes, 'N' = No. 
DocumentLedger    char(1)  NO  Indicates if the location is available to the Document Ledger. This check box must be selected for all locations at level 3. 
DropDocumentLedger    char(1)  NO  Indicates if the location is available to the Drop Document Ledger. 
VoucherLedger    char(1)  NO  Is the location is available to the Voucher Ledger? 
TransferLedger    char(1)  NO  Indicates if the location is available to the Transfer Ledger. 
Imprest_Flag    char(1)  NO  Currently not available. 
GLAccount    char(30)  YES  Indicates the GL account with which a location may be associated. 
TimeReference    char(1)  NO  If the location is a pit or cage or other. 
LocationStatus    char(1)  NO  Indicates if the PitBOSS location is active or inactive. Can be set from OASIS Administrator. 
Progressive    int  NO  Indicates if the location is a progressive. 
Type_Desc    char(30)  NO  Description of the type (e.g. table or Cage). 
TableGame_ID    char(2)  YES  ID number associated with the table game from Gen_TableGame. 
CasinoCode    char(2)  NO  Uniquely identifies a CasinoCode in the GEN_CasinoCode table.
