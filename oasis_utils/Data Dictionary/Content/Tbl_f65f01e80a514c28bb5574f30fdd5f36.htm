dbo.BB_PBTPtsCashUpTx Columns 
ColumnName  Domain  Datatype  NULL  Definition 
PBTTx_ID    int  NO  Unique ID assigned to each transaction as it is inserted into this table. This is an Identity field and the Primary Key. 
MasterTxID    int  NO  Self-referencing column for BB_PBTPTSCASHUPTX self-joins. BB_PBTPTSCASHUPTX has hierarchies of transactions, each with a MasterTxId The master's MasterTxID = its PBTTx_ID. 
SlotMast_ID    int  NO  A auto-generated system ID assigned by BlackBart when a slot machine is added to the CDS_Slotmast table.  
AuditDate    smalldatetime  NO  Audit date is an accounting reference to a "gaming day" that is in or through the audit process. Gaming day is defined by a period of 24 hours following the Slot Gaming Day Start set in OASIS Administrator. 
SlotNumber    int  NO  Slot Number refers to the number of the gaming machine. This number corresponds to the number on the machine's cabinet. 
Player_ID    int  NO  Unique ID number given to each player added via Super-Playmate. 
Revision    smallint  NO  Each time a slot machine record is opened, modified in any way, and then saved through BlackBart, the revision number is incremented by +1. 
Denomination    numeric(12, 2)  NO  The monetary value a slot machine is setup to use. 
Amount    numeric(12, 2)  NO  Personal Banker points to cash up amount. 
VoidFlag    bit  NO  Indicates whether or not the transaction was voided. 
UserID    int  NO  Id of user who created/affected the transaction. 
Source    tinyint  NO  Source of the transaction: 1=Manually added 2=Loaded 3=Adjustment. 
Comments    varchar(255)  YES  User's comments. 
Trans_ID    int  YES  References eft_Trans.Trans_Id. Transactions are loaded/unloaded into/from BB_PBTPTSCASHUPTX from eft_Trans with a Source = 2 and TransCode = 'H' (CEP Out (CEP Credits Up)).
