dbo.CDS_JFPendHist 
Logical Entity Name  CDS_JFPENDHIST 
Default Table Name  CDS_JFPendHist 
Physical Only  NO 
Do Not Generate  False 
Owner  dbo 
Definition  This table contains a history of each transaction that was at one time a transaction in the CDS_JFPEND table. What CDS_JFPENDHIST contains is exactly how the transactions from the CDS_JFPEND table were removed from that table.

Casino_ID    tinyint  NO  Unique casino ID used in conjunction with mulit-casino or multi-site casinos. Default: dbo.DEF_CASINOID. 
JFPend_ID    int  NO  Unique ID assigned to each transaction as it is inserted into this table. This is an Identity field and the Primary Key. 
TransType    char(1)  NO  Identifies the transaction type which may be either 'J' for jackpot or 'F' for fill. **Note:** 'F' is also used to indicate transaction types for PMFills. 
SlotID    int  NO  Casino-specific number assigned to a slot machine. Generally, used to seperate machines based on denomination. For example, numbers in the 1000's represent nickel machines, numbers in the 2000's represent quarter machines, and so on. 
HistDateTime    datetime  NO  A unique timestamp for when a transaction was inserted into the table. 
Status    char(1)  NO  Status shows the method of how a transaction was removed from the CDS_JFPEND table. 'R' = the transaction was processed as normal, 'P' = a transaction was purged, 'D' = the transaction was deleted via the Jackpot/Fill TX display by using the Delete key. 
JackpotFill_ID    int  NO  Unique ID assigned to each transaction as it is inserted into this table. This is an Identity field and the Primary Key.
