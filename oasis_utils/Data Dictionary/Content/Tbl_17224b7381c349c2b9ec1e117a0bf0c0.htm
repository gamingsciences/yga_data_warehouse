Definition  This table keeps track of the number and lenght of trips for the players. The StatTrip_ID is the link to CDS_STATTRIPDETAIL, which contains the detail activity information of the Player or group for that trip.

dbo.CDS_StatTrip Columns 
ColumnName  Domain  Datatype  NULL  Definition 
StatTrip_ID    int  NO  The StatTrip_ID is the link to cds_stattripdetail, which contains the detail activity information of the player for that trip. 
PlayerID    int  NO  Uniquely identifies a Player in the CDS_Player table. 
StartGamingDate    date  NO  The gaming date when the transaction was done. 
EndGamingDate    date  NO  The gaming date when the transaction was done. 
TripType    char(1)  NO  The column only 3 possible values. 'C' = The current trip, 'L' = The previous trip, 'X' = neither the current or previous trip. 
Sessions    int  NO  Number of sessions. 
IDType    AS ('P')    Previously indentified the account type. 'A' = Account, 'P' = Player, 'G' = Group. in version 12.4 and forward, only type P will be used. This column was maintained for backward compatibility and changed to a computed column where P will always be inserted as the value. The table scripts will only keep type P records during an install/upgrade. The column may be removed in a future release. 
Meta_ID    AS ([PlayerID])    The Meta ID was previously either a Player ID or a Group ID. Since the IDType column, which allowed a differentiation between the two, will always only support a PlayerID going forward, this column was changed to a computed column where the Meta_ID will always equal the PlayerID value inserted. The column was kept for backwards compatibility and may be removed in a future release.
