dbo.JF_Config Columns 
ColumnName  Domain  Datatype  NULL  Definition 
Timestamp    datetime  NO  The date and time when the record was inserted or last updated. 
Casino_ID    tinyint  NO  Unique casino ID used in conjunction with mulit-casino or multi-site casinos. Default: dbo.DEF_CASINOID. 
IsDispatchSite    char(1)  NO  Flag that indicates if Jackpots/Fills can be dispatched to another booth/window combination than the current one in which it's being processed on. 'Y' = Yes, 'N' = No. 
NumberByBooth    char(1)  NO  When searching for tickets, if this value is set to true, the results will be sorted by booth/window combo. 
PollingDelayTime    int  NO  Number of seconds between automatic refreshes (polling) of the jackpots and fills transaction grids. 
BoothWindowTicketMask    char(20)  NO  Mask to specify how Jackpot tickets look. For exampe, BBBB-WW-TTTTTT gives a ticket with 4 numbers for the booth, 2 numbers for the window and 6 numbers for the a unique ticket number (e.g. 0012-04-442370). 
AllowRemotePrint    char(1)  NO  For ticket printing, if configured to true 'Y' then the user can choose a network (non local) printer to send the ticket to instead of sending it automatically to the local printer. 
FilterByFloorCard    char(1)  NO  If set to true 'Y', then only Jackpots and Fills with the same floor card as currently inserted are displayed. 
MaxFillCount    smallint  NO  The maximum number of fills that can be performed without an override. 
UseMaxFillCutoffTime    char(1)  NO  Indicates if Jackpot/Fill should use the number of minutes specified by MaxFillCutoffTime (otherwise it used 1440 minutes (1 day). 'Y' = Yes, 'N' = No. 
MaxFillCutoffTime    int  NO  Number of minutes to use before resetting the fill count used to determine if a fill can be performed without an override. For example, if this value is 60, then Jackpot/Fill count the number of fills performed in the last 60 minutes and see if that's greater than the MaxFillCount. 
MachineLockoutMinutes    smallint  NO  Number of minutes immediately after a jackpot or fill during which an override is required in order to process the transaction. 
AllowTicketReprint    char(1)  NO  If set to "Y", then allow reprinting of Jackpot tickets, Taxforms, and Fills. 
AllowEditFillAmount    char(1)  NO  If set to "Y", allows editing of the amount entered for a fill. Otherwise the default is used and cannot be changed. 
AllowDeleteTransactions    char(1)  NO  If set to "Y", allows deletion of pending jackpot or fill transactions. 
KeypadRequiredForFills    char(1)  NO  If set to "Y", will only return fills for processing where a keypad amount has been entered. 
EnableW2G    char(1)  NO  If set to "Y", enables entry of W2-G forms when processing a jackpot that exceeds the W2-G amount, or through a manual taxform entry. 
W2GTriggerAmount    numeric(12, 2)  NO  Monetary amount that will trigger Jackpot/Fill to display a W2-G form when processing a jackpot that meets or exceeds this amount. 
AllowUnclaimedJackpots    char(1)  NO  If set to 'Y', then allow the user to mark a jackpot as unclaimed and store that information in JF_UNCLAIMED. 
OverrideNoSerialAmount    char(1)  NO  If amount from the machine is zero (0) and this is set to 'Y', then set an override. 
OverrideNoSerialOrKeypad    char(1)  NO  If both the amount from the machine is zero (0), and the keypad amount is (zero) 0 and this value is set to 'Y', then set an override. 
ExceptionNoKeypadAmount    char(1)  NO  If amount from the keypad is zero (0) and this is set to 'Y', then set an override. 
UseKeypadAmountFirst    char(1)  NO  For fills and jackpots, if this is set to 'Y', display the keypad amount if it's non-zero else display the machine amount. If set to 'N', display the machine amount if it's non-zero else display the keypad amount. 
SecureStations    char(1)  NO  If this is set to 'Y' and the workstation has an attached card reader, require that a floor card be inserted in the reader for the user to be able to use the program. 
AllowPMFills    char(1)  NO  If this is set to 'Y', display the Prevenaitive Maintainence (PM) fills grid in the transaction display. 
EnablePromoJP    char(1)  NO  If set to 'Y', allow the user to associate a promotional value (in addition to the normal value) to the jackpot when processing it. 
Enable1099    char(1)  NO  If set to 'Y', enables entry of 1099 forms when processing a jackpot that exceeds the 1099 amount, or through a manual taxform entry. 
Enable1042S    char(1)  NO  If set to 'Y', enables entry of 1042S tax forms through a manual taxform entry. 
MinHopperLevel    smallint  NO  Unused field. 
PMFillPollingTime    int  NO  Number of seconds in between polling attempts for updated PM data. 
RoundJackpot    char(1)  NO  Round normal jackpots (non-progressive). 'C' = To the nearest coin, 'D' = To the nearest dollar. 
RoundProgressiveJackpot    char(1)  NO  Round progressive jackpots. 'C' = To the nearest coin, 'D' = To the nearest dollar. 
UseDualAmountAuth    char(1)  NO  Loaded by Jackpot/Fill, but not used. 
AllowEditJackpotAmount    char(1)  NO  If set to 'Y', allow editing jackpot amounts (not applicable for manual jackpots). 
ManualTaxFormPostToPlayerStats    char(1)  NO  Default value to post manual tax form information to player stats. This can be overriden in v. 11.4.1 on the form itself. 
InsertCardRequired    char(1)  NO  Security option that requires that a valid card be in the card reader when the user presses the OK button on the authorization form. 
PinRequired    char(1)  NO  Security option that requires that the user enter a PIN when the user presses the OK button on the authorization form. 
UserNamePasswordRequired    char(1)  NO  Security option that requires the user to enter a password when the user presses the OK button on the authorization form. 
UserPINReqAtSlot    char(1)  NO  Used by the Poller to determine if the user must enter a PIN at the slot machine. 
UseSecurity    char(1)  NO  Loaded by Jackpot/Fill, but not used. 
TriggerAmt1099    numeric(12, 2)  NO  If the promo amount meets or exceeds this value, then Jackpot/Fill will prompt for a 1099 tax form entry. 
EmpCardInUpdates    char(1)  YES  Indicates if updates are allowed on Employee Card In on Jackpot/Fill processing. 
RoundJPUpDown    varchar(4)  YES  Round the Jackpot Up/Down 
JPDenominationType    varchar(12)  YES   
RoundProgJPUpDown    varchar(4)  YES  Round the Progressive Jackpot Up/Down 
ProgJPDenominationType    varchar(20)  YES  Round progressive jackpot type. Machine denomination or Dollar. 
JPDenomination    numeric(12, 2)  YES   
EnablePrinterTilt    char(1)  NO  Indicates whether or not printer tilt is enabled. 
ProgJPDenomination    numeric(12, 2)  YES  Progressive jackpot rounding denomination used when the rounding option is enabled. 
KeypadAmtCompareMachAmt    bit  NO  Indicates if the keypad amount should be compare to the machine amount or rounded amount when checking for overrides. 
EnforceAuthKeyedBack    bit  NO  If true uses the key backed jackpot amount for authorization purposes otherwise a zero amount. 
PlayerInfoSource    tinyint  NO  Indicates the source where the player information should be retrieved from when processing a tax form. 
AllowKeyBackJP    bit  NO  Allow key backed jackpots to the machine credit meters.
