dbo.BB_FillTx Columns 
ColumnName  Domain  Datatype  NULL  Definition 
FillTx_ID    int  NO  Unique ID assigned to each transaction as it is inserted into this table. This is an Identity field and the Primary Key. 
MasterTxID    int  NO  The transaction id this entry is an edit of. If this is the original entry for a given transaction then the TxID will equal the MasterTxID. Generally this table should be grouped by MasterTxID unless detail of every edit to every transaction is wished to be quarried. 
JackpotFill_ID    int  YES  Unique ID assigned to each transaction as it is inserted into this table. This is an Identity field and the Primary Key. 
Booth_ID    smallint  YES  A unique numeric value from 0 - 99 to identify a particular Jackpot/Fill workstation. 
SlotMast_ID    int  NO  A auto-generated system ID assigned by BlackBart when a slot machine is added to the CDS_Slotmast table.  
AuditDate    smalldatetime  NO  Audit date is an accounting reference to a "gaming day" that is in or through the audit process. Gaming day is defined by a period of 24 hours following the Slot Gaming Day Start set in OASIS Administrator. 
Revision    smallint  NO  Each time a slot machine record is opened, modified in any way, and then saved through BlackBart, the revision number is incremented by +1. 
SlotNumber    int  NO  Slot Number refers to the number of the gaming machine. This number corresponds to the number on the machine's cabinet. 
Denomination    numeric(12, 2)  NO  The monetary value a slot machine is setup to use. 
Amount    numeric(12, 2)  NO  Dollar amount of this transaction. 
IsMiscPay    char(1)  NO  Unused field. Defaults to 'N'. 
TicketNum    int  YES  System-generated unique number used to identify each each Jackpot/Fill ticket. 
VoidFlag    bit  NO  Bit flag to indicate if this transaction was voided. 
UserID    int  NO  The ID of the user who created this transaction. 
Source    tinyint  NO  Used to determine the souce of this record. (1=Manual Entry, 2=Loaded, 3=Edit to existing) 
Comments    varchar(255)  YES  User entered comments for this entry.
